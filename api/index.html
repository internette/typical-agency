Intro

Welcome to Typical.Agency's API documentation page. This is a RESTful api built by Antoinette Janus that powers the website typical.agency, a project created also by Antoinette Janus.

Why should I use this?

Ever been stuck in with a prototype or wireframe and needed dummy content, content that expands beyond that of "Lorem ipsum dolor sit amet..."? Wonder how all those sponsor logos that your client wants to showcase will look when actually implemented? Need to sell your design, but don't want to spend hours searching for work-appropriate content? Look no further! This API provides Creative Commons images, buzzwords, and over twenty recognizable logos.

Endpoints

The Typical.Agency REST API contains 6 main endpoints: bios, clients, images, colors, people, and words. Every API call begins with http://typical-agency.com. Below are the endpoints and the outputs.

/bios

Type: GET

This endpoint provides generic biographies for any company as well as a correlating image. These can be used as slogans, mottos, and simple About blurbs.

/colors

Type: GET

This endpoint provides generic colors and their respective names. The colors chosen are all of the colors from the Google Material Design scheme, including their names.

/clients

Type: GET

This endpoint provides a diverse list of well-known companies, their logos, and their URLs. Some of the companies includes are Facebook, McDonalds, Nike.

/images

Type: GET

This endpoint provides generic images of two categories: projects and backgrounds. The images come in 4 sizes in terms of their widths:

  1. Original
  2. Large (1024px)
  3. Medium (800px)
  4. Small (500px)

The images are all from either Creative Commons or Pexels, both allowing for modifications of images and any form of usage.

The endpoint can be filtered into the following endpoints, if you want limited results (and a faster return time):

  • /images?category=backgrounds
  • /images?category=projects

/images?category=backgrounds

Type: GET

This endpoint can be used to return background-type images, such as desk tops and workplaces.

/images?category=projects

Type: GET

This endpoint can be used to return project-type images. The types of images returned are divided into the following categories:

  • Branding ?category=backgrounds&type=brand
  • Web Sites ?category=backgrounds&type=web
  • Marketing ?category=backgrounds&type=marketing

Or, if you like a particular image, you can always us the query ?id=[ID NUMBER] to retrieve only the one image.

/people

Type: GET

This endpoint provides generic images of individuals, cropped in a "unique" way. The images come in 3 sizes:

  1. Large (800x800px)
  2. Medium (500x500px)
  3. Small (300x300px)

/words

Type: GET

This endpoint provides three different kinds of single words that are rather generic:

  • Buzzwords ?category=buzzwords
  • First Words ?category=firstwords
  • Second Words ?category=secondwords

/words?category=buzzwords

Type: GET

This endpoint can be used to return different types of buzzwords, including tech and business.

  • Tech ?category=buzzwords&type=tech
  • Business ?category=buzzwords&type=business

/words?category=firstwords

Type: GET

This endpoint can be used to return different types of generic tech words found in company names.

/words?category=secondwords

Type: GET

This endpoint can be used to return different types of generic business words found in company names.

Things to come

Below are a future plans related to the api, in no particular order.

  • Pagination

    Currently, the API returns all results, which can bog down a return on the client side. Pagination helps on the return, limiting the retrieval content but speeding up the return. As the content grows, this will help both on the server and client side.

  • Limits

    Currently, the API returns all results, which can bog down a return on the client side. In combination with pagination, limits help in returning even less data, speeding up the client side retrieval.

  • Skips

    Currently, the API has no skip function because there is no limit query. When limits are in place, skips will allow for the user to combine their query with limits to retrieve not only the first x results, but x results at any position.

  • Renaming first and second

    First and Second Words (?category=firstwords and ?category=secondwords) were appropriate for the original Typical.Agency, when it was a Codepen, but now the project has progressed. The API and names will reflect this in future updates.
